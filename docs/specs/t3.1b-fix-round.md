# UI-FIX-2 — punch list from visual judgment of the reskinned dashboard

Six surgical fixes from a side-by-side audit of the running app against `/Users/sunny/Desktop/Projects/orange-replay/design-final.html` (read-only reference — NEVER modify it). Scope: edit ONLY `apps/dashboard/**`. Do NOT run git. Implement EXACTLY these items, nothing else.

1. **Pluralization**: error pills render "1 errors" — must be "1 error" / "N errors" (singular at 1). Fix everywhere StatusPill error counts render (sessions table Status column, session-detail chips). "N rage" stays unpluralized ("1 rage", "2 rage") — matches the reference.

2. **Duration format**: durations render "10m 12s" / "22s" — the reference uses mono colon format. Format as `m:ss` (e.g. `10:12`, `3:04`, `0:22`; ≥60 min → `h:mm:ss`). Apply in the sessions-table Duration column AND the session-detail Duration stat. Update/extend the existing formatter in `src/lib/format.ts` and its unit tests if present.

3. **Size format**: sizes render "512 KB" / "1.1 MB" — reference uses compact `512K`, `145K`, `1.1M`, `3.3M` (no space, single letter, one decimal for M below 10, none at ≥10). Sessions-table Size column (and anywhere else bytes render).

4. **Session meta line city**: sessions-table Session column meta line renders "🇺🇸 US · 23 clicks" — the reference shows the CITY: "🇺🇸 San Francisco · 23 clicks". Use `city` from the API item when non-empty, falling back to the country code. Keep the flag emoji from the country code. Session-detail chips stay as country-code chips (unchanged).

5. **Load more button**: currently light-filled (primary). It must be the neutral bordered style: `bg-card border border-border text-foreground text-[12.5px] font-medium rounded-lg` (the design's `.btn` default). Keep spinner+disabled while fetching and hidden when `nextBefore` is null.

6. **Login rejection must not lose state**: typing a bad token and submitting currently navigates away, bounces back to `/login?reason=unauthorized`, CLEARS the field, and shows no error. Required flow: on Continue, first validate the token directly (call the worker health/auth-checked endpoint with `Authorization: Bearer <typed token>`) WITHOUT persisting or navigating; while validating show the button's loading spinner; on failure keep the typed value in the field and show `text-[13px] text-danger` below the field: "That token was rejected. Check DEV_API_TOKEN and try again."; only on success persist the token and navigate to sessions. Additionally, when the page loads with `?reason=unauthorized` (redirect from an expired/invalid stored token), show the same danger-styled message. Keep all existing styling (input-group, eye toggle, helper text) exactly as is.
