# T3.4 — Live tab (sessions-list completion)

The sessions list itself (filters, table, row nav) shipped in T3.1b — this task adds the **Live** surface fed by the T3.2 presence registry. Read `docs/design-language.md` first (tokens, `.lit`, pills, shell) and `design-final.html`'s "Live now" panel — the live rail must match that mock treatment exactly. Scope: edit ONLY `apps/dashboard/**`. Do NOT run git. Do NOT modify the design mocks.

Backend contract (already live): `GET /api/v1/projects/:projectId/live` (bearer auth) → `{ sessions: [{ session_id, started_at, last_seen, entry_url, country, city, browser, os, device, duration_ms }] }`, sorted by `last_seen` DESC.

## 1. Route + tab

- New route `/projects/:projectId/live` inside the app shell, wired like sessions/settings.
- Tier-2 nav gains a **Live** tab between Sessions and Settings — exact same tab treatment (13px, `py-[10px] px-[13px]`, `border-b-2 -mb-px`, active = `text-foreground border-amber font-medium`).

## 2. Live page

- Page header per design-language: title "Live" (18px semibold tracking-tight) with inline subtitle `text-[12px] text-dim ml-[10px]`: "Sessions happening right now."
- ONE `.lit rounded-lg` panel, padding `px-[18px] py-4`:
  - **Panel title row** (`flex justify-between items-baseline mb-[14px]`): left `text-[13px] font-semibold` "Live now"; right `text-[11.5px] text-dim` "updates every 5s".
  - **Rows** (`flex items-center gap-[10px] py-[9px]`, separated by `border-b border-dashed border-dash`, last row borderless):
    1. Pulsing dot: `size-[7px] rounded-full bg-success flex-none` with a 1.8s keyframe pulse (opacity dips to 0.25 at 50% — add `@keyframes pulse` to index.css if not present; respect `prefers-reduced-motion: reduce` by disabling the animation).
    2. Text block: entry page path `text-[12.5px] font-medium` (truncate, max ~340px); below it `text-[11.5px] text-dim`: `{flag} {city} · {browser}` (regional-indicator flag helper already exists; city falls back to country code; omit `· {browser}` when browser unknown).
    3. Right (`ml-auto`): elapsed time `font-mono text-[11.5px] text-muted-foreground tabular-nums`, formatted `m:ss` from `duration_ms` via the existing `formatDuration`.
  - **Whole row navigates** to the session detail route — same pattern as the sessions table rows (`role="link"`, tabIndex 0, Enter/Space, cursor-pointer, hover `bg-[#141419]`, focus ring inside).
- **Polling**: fetch on mount, then every 5s. Pause polling while `document.visibilityState === "hidden"` (visibilitychange listener; resume with an immediate fetch). Clean everything up on unmount. Re-render elapsed times only on poll (no per-second ticking).
- **States**: loading = 3 skeleton rows (existing Skeleton component, `bg-secondary`); error = existing alert treatment with a retry button; empty = dashed empty box (`border border-dashed border-dash rounded-lg`, centered `text-[13px] text-muted-foreground`): "No live sessions. Active visitors appear here within seconds."
- No "N watching" chip, no viewer counts — that data doesn't exist yet (design-language principle: nothing fake).
