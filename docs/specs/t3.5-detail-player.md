- **Left: player panel** — ONE `.lit rounded-lg overflow-hidden` card containing:
  1. **Viewport**: `aspect-video` (min-height 360px) dark stage (`bg-background`) hosting the OrangePlayer container; the replayer iframe scales to fit (rrweb replayer handles scaling; ensure `inset-0` absolute fill).      - Current time: `font-mono text-[12px] text-muted-foreground tabular-nums`.
     - **Timeline** (`flex-1 h-[26px] relative`, click + drag to seek):
       - dashed baseline: absolute full-width `border-t border-dashed border-dash` at `top-[12px]`;
       - activity ticks from the manifest `timeline` (bucketed): `w-[4px] rounded-[1.5px] bg-[#2e2e38]` bottom-anchored, height ∝ activity;

- **Right: event sidebar** — `.lit rounded-lg` panel, `px-[18px] py-4`:
  - Title row: `text-[13px] font-semibold` "Timeline" + right `text-[11.5px] text-dim` count ("N events").

  - Clicking a row seeks the player to that offset (and plays a subtle amber flash on the timeline playhead).
  - Empty: dashed empty box "No indexed events."

## 2. Wiring

- Instantiate OrangePlayer with the dashboard's API base. Private REST requests use the same-origin Better Auth session cookie; demo requests use the anonymous read-only context. Live follow mints a short-lived ticket before opening the WebSocket. Destroy on unmount. Route param changes re-create the player.
- Progress/timeline/segment/buffering/ended/error events drive the bar + states. Player `error` renders the existing alert treatment with a retry.
- **Live sessions**: if the manifest fetch 404s, check `fetchLiveSessions` for this session id; if live, render the player in follow mode (`follow()`) with a "LIVE" indicator: pulsing green dot + `text-[11.5px] text-success font-medium` "LIVE" replacing the total-duration slot; hide speed control while following. If neither manifest nor live: existing not-found treatment.
- Overlay colors: pass design tokens (amber `#f5a623`, red `#f4534e`, teal `#2dd4bf`) to the player's overlay options.
