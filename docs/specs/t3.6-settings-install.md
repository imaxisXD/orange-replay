## 1. Settings page (rebuild of settings.tsx, keeps Dev-token + Worker-health cards)

Layout: page header "Settings" + inline dim subtitle "Project configuration and keys." Cards grid (2-col on wide, stacked narrow), all `.lit rounded-lg p-5`:

1. **Capture card** — title `text-[15px] font-medium` "Capture", body 13px muted. Rows (`flex justify-between items-center py-[9px]`, dashed separators like the live rail): label + description (13px / 11.5px dim) left, control right:
   - "Sampling rate": input-group `w-[96px]` numeric percent (0–100, maps to `sampleRate` 0–1) + "%" suffix.
   - "Retention": input-group `w-[96px]` numeric days (1–365) + "days" suffix.
   - Capture toggles: four Switch rows — Heatmaps, Console, Network, Canvas (`capture.*` — amber when on).
2. **Masking card** — title "Masking". `maskPolicyVersion` shown as a neutral pill ("policy v1"). Rules editor for `maskRules[]`: each rule row = selector input-group (mono 12px, placeholder ".selector, [attr]") + action Select (mask | block) + ghost trash icon-button; "Add rule" bordered button below; empty state: dashed box "No custom rules — inputs are masked by default." Client-side guard: max 200 rules, non-empty selector.
3. **Origins card** — title "Allowed origins". Chip list of current origins (neutral pills w/ ghost × to remove) + input-group with "Add origin" button (validate: `*` or an https?:// origin). Body note 11.5px dim: "Requests from other origins are rejected at ingest."
4. **Keys card** — title "Write keys", body "Keys authenticate the SDK's ingest requests. Values are shown only where you created them." Table (per design-language table treatment): Key (mono, `key_hash` first 12 chars + "…"), Status (ok pill "active" / neutral "revoked"), Created (relative, dim). Data from the new GET keys endpoint. No create/rotate UI (backend doesn't support it yet — do not fake it).
5. **Save flow** (applies to cards 1–3): a sticky footer bar inside the settings page (`.lit` row, right-aligned) that appears only when dirty: dim "Unsaved changes" + bordered "Discard" + primary (light-filled) "Save changes" with spinner. Save = one PUT with the full update payload; success → subtle ok pill "Saved" fading after 2s; failure → danger text with the server error. Version conflict note: response's `version` replaces local.
6. Keep the existing **Dev token** and **Worker health** cards as-is at the bottom.

## 2. Install page

New route `/projects/:projectId/install` + nav-2 tab "Install" (after Settings; same tab treatment).

- Header: "Install" + subtitle "Add the snippet and verify events arrive."
- **Snippet card** (`.lit p-5`): title "Loader snippet"; body 13px muted "Paste before `</head>`. The loader is under 2 KB and defers everything else."; then a `bg-secondary border border-border rounded-lg p-4` block with the snippet in `font-mono text-[11.5px] leading-relaxed` (wrap, `overflow-x-auto`), generated at render time via `buildLoaderSnippet` from `@orange-replay/sdk` (import the loader module; if the package.json doesn't export that subpath, add the export mapping — 1 line in `packages/sdk/package.json`). Config: `bundleUrl` = placeholder `https://YOUR_HOST/or-recorder.js`, `init: { key: "YOUR_WRITE_KEY", ingestUrl: "https://YOUR_HOST" }`. Copy icon-button top-right of the block (check swap 1.5s, tooltip "Copy snippet").
- **Verify card** (`.lit p-5`): title "Live verify"; poll `GET install-status` every 3s while this page is visible (same visibility-gating pattern as the Live tab):
  - `firstEventAt === null`: pulsing amber dot + 13px "Waiting for the first event…" + 11.5px dim "Open a page with the snippet installed — this updates the moment data arrives."
  - non-null: ok pill "Installed" + 13px "First event seen {relative time}" (+ absolute in `title`).
