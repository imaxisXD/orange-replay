## 1. Template mirror (`scripts/mirror-template.mjs` → `infra/template/`)

A Node script (no new deps; use node:fs/path) that DERIVES the self-host template from the canonical worker so it cannot drift:

- Copies `apps/worker/migrations/**` verbatim.
- Generates `infra/template/wrangler.jsonc` FROM `apps/worker/wrangler.jsonc`: same DO classes/migrations/queues/cron, but with self-host-appropriate names (`orange-replay` worker name), placeholder database ids (`REPLACE_WITH_D1_ID`), a `# created by setup docs` comment per binding, and secret guidance for Better Auth, GitHub OAuth, and live tickets WITHOUT any test-only vars (`DEV_TEST_ROUTES`, `TEST_TIMINGS` must NOT appear). The original shared-token wording in this historical spec is superseded.
- Writes `infra/template/README.md` (short: what this is, link to docs/self-host.md, Deploy-button placeholder note "button wiring lands when the public template repo is published").
- Emits a manifest `infra/template/.mirror-manifest.json` = { sourceHash: sha256 of the inputs, generatedAt: build-arg-injected ISO string passed via CLI flag `--stamp` (do NOT call Date.now() default — accept the flag; default "unstamped") } so CI can dry-run compare.
- `--check` mode: regenerates in a temp dir and exits non-zero with a diff summary if `infra/template/` is out of date (this is the CI dry-run).
- Add `"mirror-template": "node scripts/mirror-template.mjs"` to root package.json scripts.
- Unit test (node env, `scripts/mirror-template.test.mjs` or under a tests dir consistent with repo layout): generates into a temp dir; asserts placeholders present, test-only vars absent, migrations copied, `--check` passes right after generation and fails after tampering.
