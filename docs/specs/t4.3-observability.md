## Checklist

1. **Coverage — one wide event per unit of work**: enumerate every unit of work (each HTTP route incl. 404/health/OPTIONS preflight, every DO entrypoint — fetch routes, webSocketMessage/Close/Error, alarms — every queue message, the sweeper cron, every presence DO request). Table each with its event name or FINDING. Known previously-added ones: `ingest.batch`, `do.append`, `do.alarm`, `do.finalize`, `do.presence`, `api.request`, `http.not_found`, live-connect. Gaps to check specifically: `webSocketClose`/`webSocketError` handlers (viewer disconnects should emit or be folded into a connect/disconnect event), queue consumer per-message vs per-batch (contract says per message), sweeper cron run event with rows/objects deleted counts.
2. **Emitted in `finally`**: verify each emit sits in a finally block reachable on throw; any early-return path that skips the emit is a finding.
3. **Base fields**: ts, service, event, request_id, outcome, status_code?, duration_ms, version on EVERY event (logger enforces most — verify `version` is populated from a real build/version value, not hardcoded "dev" in production paths; if only "dev" exists, wire an `APP_VERSION` var with "dev" default and document in the template mirror inputs if wrangler.jsonc changes — keep template mirror --check green by re-running the mirror).
4. **Identity fields**: project_id/org_id/session_id/tab/seq present where known. Spot-check ingest, DO append, finalize, consumer.
5. **request_id propagation**: minted at edge (UUIDv7), propagated via x-or-request-id into SESSION DO (append + live), PRESENCE DO, queue messages (FinalizeMessage.requestId), and used by the consumer's events. Add an integration test asserting one request id flows edge→DO→queue→consumer across a full session lifecycle (extend the existing e2e-style integration test rather than inventing new harness).
6. **No stray console**: `console.*` only inside logger.ts. Grep-verify across apps/worker and packages/shared.
7. **`observability.enabled`**: ensure wrangler.jsonc (canonical + template via mirror) enables Workers observability/logs (`"observability": { "enabled": true }`) so wide events are actually collected on real deploys.
8. **Event-size sanity**: no unbounded fields in events (error messages truncated; no payload bytes, ticket/token material, or full URLs with query strings — strip query from any logged URL).
