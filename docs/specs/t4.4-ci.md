# T4.4 — CI: GitHub Actions

Create the repo's CI so every gate we run by hand runs on push/PR. Scope: `.github/workflows/**` (new), plus at most tiny `package.json` script aliases if a job needs one. Do NOT run git. Nothing may deploy or touch real Cloudflare resources; no secrets referenced anywhere.

## Workflow `ci.yml` (on: push to main + pull_request)

Toolchain notes: pnpm via corepack (version from package.json packageManager field if present, else lockfile-compatible); the repo pins `vite-plus` as a dependency — run it via `pnpm exec vp` (verify `node_modules/.bin/vp` exists after install; do NOT rely on a global install). Cache the pnpm store. Node version: match what the repo/vp expects (check .node-version/engines; else current LTS). Set `CI=true` implications for pnpm install (frozen lockfile is correct in CI).

Jobs (parallel where independent, each with `timeout-minutes`):

1. **check** — `pnpm install --frozen-lockfile` + `pnpm exec vp check`.
2. **test** — full `pnpm exec vp test` (workerd integration included; ubuntu runner works). Keep `CLOUDFLARE_LOAD_DEV_VARS_FROM_DOT_ENV=false` in the test config so a developer's local `.env` can never change the fails-closed test.
3. **budgets** — build the SDK and run its budget gate (`packages/sdk/scripts/check-budgets.mjs` — read how it's invoked locally, likely a package script; wire exactly that).
4. **e2e** — `npx playwright install --with-deps chromium` then run the fixture suite from `fixtures/demo-site`. Upload the playwright report/trace as an artifact ONLY on failure.
5. **mirror** — `node scripts/mirror-template.mjs --check`.
6. **build** — dashboard production build (`vp build` in apps/dashboard) to catch build-only breakage; pack the publishable packages if `vp pack` works without the vendored-fork dts hang (packages/rrweb-fork dts is disabled — verify with a timeout; if pack is flaky in CI, scope this job to the dashboard build and note it).

Conventions: concurrency group cancelling superseded runs on the same ref; minimal permissions (`contents: read`); no third-party actions beyond `actions/checkout`, `actions/setup-node`, `pnpm/action-setup` (or corepack), `actions/cache`, `actions/upload-artifact`.

## Verification you CAN do (and must)

You cannot run GitHub Actions. Instead: (a) YAML must parse (validate with node's yaml via a one-off script or python's yaml if available); (b) EVERY command line the workflow runs must be executed locally verbatim and pass (list each with its result); (c) `pnpm exec vp --version` works from a clean-ish shell. Report any command that behaves differently locally vs what CI will see (e.g. HOME-dependent wrangler paths — CI HOME is writable, fine).

## Definition of done

`vp check` green (the workflow files shouldn't affect it, but run it); all workflow commands proven locally; report the job table with local-run results.
