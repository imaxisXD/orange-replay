# T5.1 — Replay-to-Repro: from a session to a failing Playwright test + agent-ready failure bundle

## Goal (what exists when this is done)

On any finalized session's detail page, a **"Repro"** button opens a panel with three copyable artifacts, generated **entirely in the viewer's browser** (zero new server surface, zero payload inspection server-side — this is a privacy talking point, preserve it):

1. **A runnable Playwright spec** that replays the user's exact path (navigation, clicks, typed inputs as placeholders, viewport) and **asserts the recorded page error does not occur** — i.e. it FAILS while the bug exists and PASSES once fixed. A regression test, not a transcript.
2. **A failure bundle (JSON)** — machine-readable: ordered steps with selectors and timestamps, error signatures, DOM context around the failure, environment. Designed to be pasted into (or fetched by, see T5.2) a coding agent as complete debugging evidence.
3. **A markdown bug report** rendering of the same bundle for humans/issue trackers.

**The demo that must work at the end** (this is the acceptance scenario): record a session on the demo site clicking "Trigger TypeError"; open it in the dashboard; click Repro; paste the generated spec into `fixtures/demo-site/tests/`; `npx playwright test` runs it; it reproduces the click path against the demo site and FAILS on the pageerror assertion (because the demo bug "exists"). The generated bundle names `[data-throw-error]`-region selector, the TypeError message, and the step sequence.

## Input data contracts (read these files first; do not re-invent shapes)

- Decoded rrweb events: `ReplayEvent = { type: number; data: unknown; timestamp: number }` (see `packages/player/src/types.ts`). Relevant rrweb constants (define locally in `repro/`, do not import rrweb): EventType Meta=4, FullSnapshot=2, IncrementalSnapshot=3; IncrementalSource Mutation=0, MouseInteraction=2, Scroll=3, Input=5; MouseInteraction Click type=2. Verify these numeric values against `packages/rrweb-fork/src/vendor/rrweb/types` and the fixtures under `packages/player/tests/` before hardcoding — cite where you confirmed them in the report.
- FullSnapshot data: serialized node tree — nodes `{ id, type, tagName?, attributes?, childNodes[], textContent? }` (rrweb snapshot format; element type=2, text type=3).
- Mutation data: `{ adds: [{parentId, node}], removes: [{id}], attributes: [{id, attributes}], texts: [{id, value}] }`.
- MouseInteraction data: `{ type, id, x, y }` where `id` is the rrweb node id.
- Input data: `{ id, text, isChecked }` — **text is already masked at capture (`*`s) for inputs; treat every input value as masked**.
- Meta data: `{ href, width, height }` (href is scrubbed as of `5d2231b`).
- Manifest timeline (`IndexEvent { t, k, d?, m? }`, kinds include `click | error | rage | nav`): error events carry the truncated message in `d`. Session meta from `SessionManifest` (browser/os/viewport via attrs + Meta).
- How to obtain decoded events in the dashboard: the player already holds them; expose a getter `OrangePlayer.getDecodedEvents(): ReplayEvent[]` (add it — trivial accessor over `this.events`) plus `getManifest()`. The panel consumes the LIVE player instance; do not re-fetch/re-decode.

## Module designs

### 1. `virtual-dom.ts` — reconstruct queryable DOM state from events

```ts
export class VirtualDom {
  applyFullSnapshot(data): void; // rebuild id → VNode map
  applyMutation(data): void; // adds/removes/attributes/texts
  node(id): VNode | undefined;
  ancestors(id): VNode[]; // leaf → root
  outerHtml(id, maxChars): string; // reconstructed, attributes sorted, text included, TRUNCATED
}
export interface VNode {
  id: number;
  tag?: string;
  attrs: Record<string, string>;
  text?: string;
  parentId?: number;
  childIds: number[];
}
```

Feed it events in timestamp order up to any point → DOM state at that point. Text nodes fold into parent `text`. Handle re-snapshots (Meta+FullSnapshot mid-stream = reset). Removes cascade to descendants. This class is the foundation for selectors and failure context.

### 2. `selector.ts` — robust Playwright locator from a VNode chain

`buildLocator(vdom, nodeId): { locator: string; confidence: "high"|"medium"|"low"; comment?: string }`
Preference order (first that yields a match-worthy value):

1. `data-testid`/`data-test`/`data-cy` → `page.getByTestId(...)` (high)
2. `id` attr, UNLESS it looks generated (matches `/\d{3,}|[a-f0-9]{8,}|^:/`) → `page.locator('#...')` (high)
3. `aria-label` → `getByLabel` (high); `role`+accessible-name-ish text → `getByRole(role, { name })` (high)
4. Element text if 2–30 chars after trim (from VNode text) → `getByText('...', { exact: true })` (medium)
5. CSS path fallback: walk ancestors up to 4 levels, `tag.class1.class2` per level with `:nth-of-type()` only where a sibling of the same tag exists (low; attach comment "verify selector").
   Escape quotes; classes that look utility-generated (contain `[` `:` or >20 chars) are skipped. Always emit `.first()` NEVER — instead emit a `// NOTE: selector may be ambiguous` comment when fallback tier is used (Playwright strict mode surfacing ambiguity is a feature for the fixer).

### 3. `steps.ts` — event stream → semantic step list

`extractSteps(events, timeline, opts): ReproStep[]` where

```ts
type ReproStep =
  | { kind: "goto"; url: string; ts: number } // first Meta href (path only — origin becomes a BASE_URL param)
  | { kind: "viewport"; width: number; height: number; ts: number }
  | { kind: "navigate"; url: string; ts: number } // later Meta/nav index events (SPA or MPA — assert URL, don't goto)
  | { kind: "click"; nodeId: number; locator: LocatorResult; ts: number }
  | { kind: "fill"; nodeId: number; locator: LocatorResult; masked: true; ts: number } // collapse consecutive Input events on same node into one
  | { kind: "scroll"; ts: number } // recorded but SKIPPED in codegen v1 (comment emitted)
  | { kind: "wait"; ms: number; ts: number } // only for gaps > 5s, capped at 2s in codegen, comment shows real gap
  | { kind: "error"; message: string; ts: number }; // from timeline error index events
```

Rules: resolve click/fill selectors against VirtualDom state AS OF that event's timestamp (apply events incrementally while walking — single pass). Deduplicate double-clicks <300ms apart into one step with a comment. Steps AFTER the final error event are dropped (goal is reaching the failure). Multi-tab sessions: v1 uses the tab with the most events; emit a header comment `// session had N tabs; reproducing tab with most activity` (tab attribution comes from segment batch boundaries — thread `tab` through decode metadata; if that plumbing is missing, take events as-is and note the limitation in the generated header instead — do NOT build tab-lane infrastructure here).

### 4. `playwright.ts` — codegen

`generatePlaywrightSpec(steps, meta): string` producing a complete, formatted file:

```ts
import { test, expect } from "@playwright/test";
// Generated by Orange Replay from session <id> (<browser> · <os>, <recorded date via manifest>)
// BASE_URL: set to the environment under test.
const BASE_URL = process.env.REPRO_BASE_URL ?? "http://localhost:5300";
test("repro: <error message first 60 chars | 'user path'>", async ({ page }) => {
  const pageErrors: string[] = [];
  page.on("pageerror", (e) => pageErrors.push(String(e?.message ?? e)));
  await page.setViewportSize({ width: W, height: H });
  await page.goto(`${BASE_URL}/path`);
  // …steps: await page.getByTestId("x").click();  await page.locator("#y").fill("TEST_VALUE" /* value masked in recording */);
  // …waits/comments…
  expect(
    pageErrors.filter((m) => m.includes("<error substring, escaped, first 80 chars>")),
  ).toHaveLength(0);
});
```

Error-free sessions: last assertion becomes `expect(pageErrors).toHaveLength(0)` and the title says "user path". Every fallback-tier locator line carries its comment. Output must be deterministic for identical input (no Date.now()).

### 5. `bundle.ts` + `markdown.ts`

`buildFailureBundle(events, manifest, steps): FailureBundle`:

```ts
interface FailureBundle {
  v: 1;
  session: { id; startedAt; durationMs; browser?; os?; viewport?; entryUrl? };
  errors: { ts; message; precededBy: { kind; locator?; url?; ts }[] }[]; // last ≤5 steps before each error
  steps: ReproStep[]; // full ordered list
  failureContext?: { locator; ancestors: string[]; outerHtml: string }; // last interacted node before first error; outerHtml ≤2000 chars
  limitations: string[]; // e.g. "input values masked at capture", "network/console capture not enabled"
}
```

`renderMarkdown(bundle)`: title = error message; environment table; "Steps to reproduce" numbered with locators; error section; failure-context code block; limitations footnote. Stable ordering, no timestamps-as-of-now.
